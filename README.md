# photo-rescue 废片拯救器skill · 使用说明文档

## 作品名称

**photo-rescue - 全面拯救废片的 AI 修图助手**

---

## 作品简介

photo-rescue 是一个基于 wan2.7 模型的智能照片修复技能，专注于拯救"辛辛苦苦拍却不能用"的废片。它采用 **5 层注意力残差架构**，通过废片诊断、拯救策略制定、精准提示词生成、分步修复执行和预防建议，实现对 7 类常见照片问题的自动化修复。

用户无需配置 API Key，支持通过自然语言指令或专用口令系统快速触发修复流程，是摄影爱好者和内容创作者的得力助手。

---

## 解决什么问题 / 面向什么场景

### 核心问题解决

针对摄影中常见的 **7 类废片问题**：

| 问题类型 | 典型表现 | 修复难度 |
|---------|---------|---------|
| **路人/杂物乱入** | 背景出现无关人员、电线杆、垃圾桶 | 简单 |
| **姿势/表情灾难** | 眨眼、闭眼、表情扭曲、动作僵硬 | 简单 |
| **衣服/造型不搭** | 颜色冲突、褶皱、与场景风格脱节 | 简单 |
| **光影极差** | 逆光死黑、面部过曝、阴阳脸 | 中等 |
| **构图失衡** | 歪斜、主体过小/过满、地平线切割脖子 | 中等 |
| **缺少关键人物** | 本该入镜的人没拍进去 | 中等 |
| **对焦/模糊** | 手抖跑焦、主体模糊 | 中等 |

### 面向场景

1. **旅行摄影**：清除路人干扰，修复逆光人像
2. **人像摄影**：调整表情姿势，优化服装造型
3. **合影场景**：补全缺失人物，统一光影效果
4. **产品拍摄**：去除杂物，优化构图和光影
5. **日常记录**：拯救手机拍摄的模糊、构图差的废片

## 核心调用逻辑

### 整体架构：5 层注意力残差

```
最终拯救计划 = 0.20·L1 + 0.15·L2 + 0.45·L3 + 0.15·L4 + 0.05·L5
```

| 层级 | 内容 | 权重 | 说明 |
|------|------|------|------|
| **L1** | 废片诊断 | 0.20 | 识别问题类型（7 类） |
| **L2** | 拯救策略 | 0.15 | 制定修复方案（清理/增强/重构） |
| **L3** | 提示词生成 | **0.45** | **核心执行层**，生成精准修复指令 |
| **L4** | 操作流程 | 0.15 | 步骤优先级排序 |
| **L5** | 预防建议 | 0.05 | 避坑指南 |

### 触发判断流程

```
用户输入
    ├─ 有明确指令（如"清除路人""改衣服为红色""调整姿势"）
    │   └→ 直接跳转到对应阶段，输出具体修改提示词
    │
    └─ 无明确指令（如"拯救这张废片""这张照片怎么办""帮我修修"）
        └→ 执行完整诊断流程，先给出整体修改建议
```

### 标准修复优先级

按影响力从高到低排序：

```
优先级1：光影校正（影响最大，决定照片是否可用）
    ↓ 调用 wan2.7，strength = 0.5
优先级2：清除干扰（去除路人杂物，恢复干净背景）
    ↓ 调用 wan2.7，strength = 0.6
优先级3：修复主体（姿势、表情、衣服）
    ↓ 调用 wan2.7，strength = 0.5-0.6
优先级4：补全锐化（添加人物、去模糊）
    ↓ 调用 wan2.7，strength = 0.4
```

## 使用到的 Wan 能力

### 核心能力

- **wan2.7 模型（万相 2.7）**
  - 提供方：阿里云百炼（DashScope）
  - 能力：文生图、图生图、图像编辑
  - 优势：高保真主体保持，精准局部修改

### API 调用方式

通过脚本 `scripts/wanx_image_gen.py` 统一调用 wan2.7 模型：

```bash
python scripts/wanx_image_gen.py \
  --input <输入图片> \
  --prompt "<修复指令>" \
  --output <输出图片> \
  --strength <修改强度>
```

### 参数说明

| 参数 | 必需 | 说明 | 取值范围 |
|-----|------|------|---------|
| `--input` | 是 | 原始废片路径 | 图片文件路径 |
| `--prompt` | 是 | 修复指令（英文） | 具体描述修复需求的文本 |
| `--output` | 是 | 输出图片路径 | 保存路径 |
| `--strength` | 否 | 修改强度 | 0-1，默认 0.7 |

## Prompt / 工作流 / 配置说明

### 口令系统

用户可通过**口令**快速获取对应提示词，支持**组合使用**：

| 口令 | 功能 | 示例 |
|-----|------|------|
| `/clean` | 清除路人/杂物 | 删除背景中的无关人员、垃圾桶等 |
| `/face` | 修复表情/姿势 | 睁眼、去双下巴、调整姿势 |
| `/cloth [颜色]` | 改衣服颜色/质感 | `/cloth 深蓝色` - 将上衣改为深蓝色 |
| `/light [类型]` | 修复光影问题 | `/light 逆光` - 解决逆光问题 |
| `/pose` | 调整姿势 | 站直、转身、手臂位置调整 |
| `/crop [比例]` | 裁剪构图 | `/crop 4:5` - 裁剪为 4:5 比例 |
| `/addperson [性别] [位置]` | 添加缺失的人 | `/addperson 女性 左侧` |
| `/sharpen` | 去模糊/锐化 | 修复跑焦、模糊照片 |
| `/megafix` | 一键全修复 | 综合修复严重废片 |

**组合口令示例**：

```
/clean + /light 逆光 + /face
```
→ 按注意力权重顺序输出：①光影修复 → ②清除路人 → ③修表情姿势

### 提示词生成原则

1. **具体化**
   - ❌ 错误：`"修复一下"`
   - ✅ 正确：`"删除背景中的路人，保持主体不变"`

2. **保留指令**
   - 强调`"保持主体不变"`、`"保持面部特征"`、`"保持服装"`

3. **分步修复**
   - 一次修复一个主要问题
   - 保存中间版本
   - 避免一次性过度修改

### 典型修复提示词示例

#### 问题 1：清除路人/杂物


## 注意事项

### 核心原则

1. **注意力优先级**：先修复影响最大的问题（光影 > 清理 > 主体）
2. **分步修复**：严重废片建议分步处理，避免一次性过度修改
3. **保留原始特征**：所有提示词必须强调"保持主体不变"
4. **平台适配**：根据用户使用的工具调整提示词格式

### 常见误区

- ❌ 一次性修复所有问题 → 导致 AI 处理混乱，效果失真
- ❌ 只描述"要改什么"，不说"不要改什么" → AI 过度修改
- ❌ 使用模糊词汇（如"好看一点"） → 效果不可控
- ✅ 具体、明确、带约束条件的提示词效果最佳

### 高级技巧

- **权重控制**：在 Midjourney 中使用 `(keyword:1.5)` 增强权重
- **负面提示词**：使用 `--no` 明确排除不想要的元素
- **参考图强度**：即梦 AI 建议设置 0.8-0.9，确保主体一致性
- **分图层处理**：Photoshop 中分层修复不同问题，避免相互干扰

## 预防建议

为减少未来废片产生，建议遵循以下拍摄原则：

1. **拍照前检查背景** - 避开路人、杂物
2. **连拍 3-5 张** - 选表情最好的
3. **逆光时开闪光/补光** - 避免死黑
4. **合影用广角+人像模式** - 确保全员入镜
5. **注意地平线位置** - 不要切割人物脖子
