# 废片拯救计划 · wan2.7图像生成版

## ⚠️ 重要说明

**所有图片生成必须使用wan2.7模型（阿里云百炼），严格禁止调用其他模型。**

- 模型：wan2.7（万相2.7）
- 平台：阿里云百炼（DashScope）
- 能力：文生图、图生图、图像编辑
- API：通过脚本 `scripts/wanx_image_gen.py` 调用

---

## 任务目标
- 本Skill用于：诊断照片失败原因，使用wan2.7模型生成修复后的图片，拯救"辛辛苦苦拍却不能用"的废片
- 核心能力：**废片诊断**、**wan2.7图像生成**、**口令快速调用**
- 触发条件：用户提出"拯救废片"、"修复照片"、"修改姿势"、"清除路人"、"改衣服颜色"等需求

## 核心原则：注意力残差架构

拯救废片采用**5层注意力加权架构**，按重要性分配权重：

```
最终拯救计划 = 0.20·L1 + 0.15·L2 + 0.45·L3 + 0.15·L4 + 0.05·L5
```

| 层级 | 内容 | 权重 | 说明 |
|------|------|------|------|
| **L1** | 废片诊断 | 0.20 | 识别问题类型 |
| **L2** | 拯救策略 | 0.15 | 制定修复方案 |
| **L3** | 提示词生成 | **0.45** | **核心执行层（最详细）** |
| **L4** | 操作流程 | 0.15 | 步骤优先级排序 |
| **L5** | 预防建议 | 0.05 | 避坑指南 |

## 快速开始：口令系统

用户可通过**口令**快速获取对应提示词，支持**组合使用**。

### 口令对照表

| 口令 | 功能 | 示例 |
|------|------|------|
| `/clean` | 清除路人/杂物 | 删除背景中的无关人员、垃圾桶等 |
| `/face` | 修复表情/姿势 | 睁眼、去双下巴、调整姿势 |
| `/cloth [颜色]` | 改衣服颜色/质感 | `/cloth 深蓝色` - 将上衣改为深蓝色 |
| `/light [类型]` | 修复光影问题 | `/light 逆光` - 解决逆光问题 |
| `/pose` | 调整姿势 | 站直、转身、手臂位置调整 |
| `/crop [比例]` | 裁剪构图 | `/crop 4:5` - 裁剪为4:5比例 |
| `/addperson [性别] [位置]` | 添加缺失的人 | `/addperson 女性 左侧` |
| `/sharpen` | 去模糊/锐化 | 修复跑焦、模糊照片 |
| `/megafix` | 一键全修复 | 综合修复严重废片 |

### 组合口令示例

```
/clean + /light 逆光 + /face
```
→ 按注意力权重顺序输出：①光影修复 → ②清除路人 → ③修表情姿势

```
/cloth 深蓝色 + /pose + /sharpen
```
→ 输出：①改衣服 → ②调姿势 → ③锐化

## 操作流程

### ⚠️ 核心判断逻辑（最重要，必须遵守）

**第一步：判断用户意图**

智能体必须首先判断用户输入是否包含**明确的修改指令**：

```
用户输入
    ├─ 有明确指令（如"清除路人""改衣服为红色""调整姿势"）
    │   └→ 直接跳转到对应阶段，输出具体修改提示词
    │
    └─ 无明确指令（如"拯救这张废片""这张照片怎么办""帮我修修"）
        └→ 执行完整诊断流程，先给出整体修改建议
```

#### 判断标准

**明确指令的特征**：
- 包含具体的修改动词：清除、修改、调整、改变、添加、去除等
- 包含具体的问题：路人、姿势、衣服、光影、模糊等
- 包含具体的目标：改成XX色、调整为XX姿势、添加XX人

**无明确指令的特征**：
- 泛泛的求助：拯救废片、修复照片、帮我修修
- 问题描述模糊：照片不好看、不满意、有问题
- 仅表达不满：这张照片失败了、辛辛苦苦拍的但没用

### 情况B：无明确指令 → 先诊断+整体建议

当用户没有明确指令时，**执行完整的诊断流程**：

**步骤1：废片诊断（L1）**

使用智能体的图像识别能力，识别照片存在的问题：

**常见废片原因**（7大类）：

1. **路人/杂物乱入** - 背景出现无关人员、电线杆、垃圾桶
2. **姿势/表情灾难** - 眨眼、闭眼、表情扭曲、动作僵硬
3. **衣服/造型不搭** - 颜色冲突、褶皱、与场景风格脱节
4. **光影极差** - 逆光死黑、面部过曝、阴阳脸
5. **构图失衡** - 歪斜、主体过小/过满、地平线切割脖子
6. **缺少关键人物** - 本该入镜的人没拍进去
7. **对焦/模糊** - 手抖跑焦、主体模糊

**步骤2：输出整体修改建议**

按照**注意力权重优先级**，输出诊断报告和修改建议：

```markdown
📋 废片诊断报告

## 识别到的问题（按优先级排序）

| 优先级 | 问题类型 | 严重程度 | 影响范围 |
|--------|----------|----------|----------|
| ⭐⭐⭐ | 逆光问题 | 严重 | 面部死黑，影响整体 |
| ⭐⭐ | 背景路人 | 中等 | 干扰主体，影响美观 |
| ⭐ | 姿势僵硬 | 轻微 | 可优化，但不严重 |

## 推荐修复方案

按影响力从高到低修复：

### 步骤1：修复逆光（优先级最高）⭐⭐⭐
**原因**：光影问题影响照片是否可用，必须优先解决
**提示词**：
```
Brighten face to natural skin tone (increase exposure by 1.2 stops).
Add soft fill light from front. Reduce backlight harshness.
Increase shadow detail by 40%.
```

### 步骤2：清除背景路人（中等优先级）⭐⭐
**原因**：恢复干净的背景，突出主体
**提示词**：
```
Remove all strangers from background.
Use generative fill to reconstruct clean background.
Keep main subject untouched.
```

### 步骤3：调整姿势（可选）⭐
**原因**：优化细节，提升整体效果
**提示词**：
```
Straighten spine, relax shoulders, arms hanging naturally.
Keep original expression unchanged.
```

## 请确认

请问您想：
1. 按照这个方案全部修复？
2. 只修复某个问题？（请告诉我优先修复哪个）
3. 还是有其他具体要求？
```

详细诊断指南：见 [references/diagnosis-guide.md](references/diagnosis-guide.md)

### 阶段二：制定拯救策略（L2, α=0.15）

**目标**：根据诊断结果，制定修复方案

**三块修复策略**：

```
最终输出 = 0.3·清理层 + 0.4·增强层 + 0.3·重构层
```

| 策略层 | 功能 | 适用问题 |
|--------|------|----------|
| **清理层** | 去除多余元素 | 路人、杂物、瑕疵 |
| **增强层** | 调整光影色彩 | 逆光、过曝、色彩问题 |
| **重构层** | 重塑主体结构 | 姿势、衣服、添加人物 |

**策略组合示例**：
- 废片有路人 + 逆光 → 清理层 + 增强层
- 姿势丑 + 衣服褶皱 → 重构层（双重）
- 模糊 + 构图差 → 增强层 + 重构层

### 阶段三：调用wan2.7生成图片（L3, α=0.45）⭐

**目标**：使用wan2.7模型生成修复后的图片

**⚠️ 强制要求**：所有图片生成必须使用wan2.7模型，禁止调用其他模型。

#### 图片生成方式

**方式1：图生图（废片拯救核心）**

使用脚本 `scripts/wanx_image_gen.py` 进行图生图修复：

```bash
python scripts/wanx_image_gen.py \
  --input 原始照片.jpg \
  --prompt "修复描述，如：删除背景路人，提亮面部光影" \
  --output 修复后照片.jpg \
  --strength 0.7
```

**参数说明**：
- `--input`：原始废片路径（必需）
- `--prompt`：修复指令（必需）
- `--output`：输出图片路径（必需）
- `--strength`：修改强度（0-1，默认0.7），值越小保持原图越多

智能体根据诊断结果生成修复提示词：

### 阶段四：执行修复流程（L4, α=0.15）

**目标**：按优先级顺序调用wan2.7生成图片

**标准修复顺序**（按影响力排序）：

```
优先级1：光影校正（影响最大，决定照片是否可用）
    ↓ 调用wan2.7，strength=0.5
优先级2：清除干扰（去除路人杂物，恢复干净背景）
    ↓ 调用wan2.7，strength=0.6
优先级3：修复主体（姿势、表情、衣服）
    ↓ 调用wan2.7，strength=0.5-0.6
优先级4：补全锐化（添加人物、去模糊）
    ↓ 调用wan2.7，strength=0.4
```

**执行注意事项**：
- 每完成一步，保存中间版本
- 使用不同的strength值控制修改程度
- 严重废片建议分步修复，不要一次性处理所有问题

详细操作流程：见 [references/operation-workflow.md](references/operation-workflow.md)

### 阶段五：预防建议（L5, α=0.05）

**目标**：减少未来废片产生

**常见避坑建议**：
1. 拍照前检查背景 - 避开路人、杂物
2. 连拍3-5张 - 选表情最好的
3. 逆光时开闪光/补光 - 避免死黑
4. 合影用广角+人像模式 - 确保全员入镜
5. 注意地平线位置 - 不要切割人物脖子

## 注意事项

### 核心原则
1. **注意力优先级**：先修复影响最大的问题（光影 > 清理 > 主体）
2. **分步修复**：严重废片建议分步处理，避免一次性过度修改
3. **保留原始特征**：所有提示词必须强调"保持主体不变"
4. **平台适配**：根据用户使用的工具调整提示词格式

### 常见误区
- ❌ 一次性修复所有问题 → 导致AI处理混乱，效果失真
- ❌ 只描述"要改什么"，不说"不要改什么" → AI过度修改
- ❌ 使用模糊词汇（如"好看一点"） → 效果不可控
- ✅ 具体、明确、带约束条件的提示词效果最佳

### 高级技巧
- **权重控制**：在Midjourney中使用 `(keyword:1.5)` 增强权重
- **负面提示词**：使用 `--no` 明确排除不想要的元素
- **参考图强度**：即梦AI建议设置0.8-0.9，确保主体一致性
- **分图层处理**：Photoshop中分层修复不同问题，避免相互干扰
