# 废片拯救计划 · Skill 说明文档

## 作品名称

**废片拯救计划** (photo-rescue)

---

## 作品简介

基于阿里云百炼 **wan2.7** 模型的 AI 废片拯救工具，通过深度注意力残差架构诊断照片问题并智能修复，让"辛辛苦苦拍却不能用"的废片重获新生。

**技术特性**：
- 模型：wan2.7（万相 2.7）
- 平台：阿里云百炼 (DashScope)
- 能力：文生图、图生图、图像编辑
- 凭证：开发者模式（内置 API Key，用户无需配置）

---

## 解决什么问题 / 面向什么场景

### 解决的问题

日常拍摄中常见的 7 类废片问题：

| 问题类型 | 描述 | 示例 |
|---------|------|------|
| **路人** | 背景出现陌生人物 | 旅游打卡照被路人抢镜 |
| **姿势** | 人物姿态不自然 | 合影时闭眼、歪头 |
| **衣服** | 服装颜色/款式不满意 | 想换掉拍照时的衣服 |
| **光影** | 曝光过度或不足 | 逆光脸黑、大平光无立体感 |
| **构图** | 画面比例/位置不佳 | 人物在边缘、背景太空 |
| **缺人** | 需要添加人物 | 独旅行程照没人陪 |
| **模糊** | 脱焦或抖动模糊 | 夜景拍摄手抖糊片 |

### 面向场景

- 旅游拍照：路人抢镜、表情不佳
- 人像摄影：姿势不自然、服装想换
- 活动合影：闭眼、遮挡、构图差
- 日常记录：逆光脸黑、背景杂乱
- 直播截图：画质模糊、表情尴尬

## 核心调用逻辑

### 决策树：是否需要诊断？

```
用户输入
    │
    ▼
是否有明确修复指令？
    │
    ├─ 是 → 直接生成修复提示词 → 调用 wan2.7
    │
    └─ 否 → 先诊断问题类型 → 制定整体拯救方案 → 调用 wan2.7
```

### 注意力残差架构（5 层加权）

```
最终拯救计划 = 0.20·L1 + 0.15·L2 + 0.45·L3 + 0.15·L4 + 0.05·L5
```

| 层级 | 内容 | 权重 | 说明 |
|------|------|------|------|
| **L1** | 废片诊断 | 0.20 | 识别问题类型（7 类问题分类） |
| **L2** | 拯救策略 | 0.15 | 制定修复方案（优先级排序） |
| **L3** | 提示词生成 | **0.45** | **核心执行层（最详细）** |
| **L4** | 操作流程 | 0.15 | 步骤优先级排序 |
| **L5** | 预防建议 | 0.05 | 避坑指南 |

### API 调用流程

```
用户请求 → 智能体诊断 → 生成提示词 → wan2.7 API → 下载结果
                │            │
                ▼            ▼
           问题分类    图像编辑提示词
                │
                ▼
         调用 wanx_image_gen.py
                │
                ├─ 图生图：原图 + 提示词 → 修复后图片
                │
                └─ 文生图：纯提示词 → 创意图片
```

## 使用到的 Wan 能力

### 1. 图像编辑 (Image-to-Image)

**核心能力**：基于原图和文本描述进行局部/全局修改

```python
# 调用方式
wanx_image_gen.py --input original.jpg --prompt "移除背景路人" --output fixed.jpg
```

**适用场景**：
- 移除路人/杂物
- 调整姿势/表情
- 更换衣服/颜色
- 修复光影/构图

### 2. 文本生成图像 (Text-to-Image)

**辅助能力**：纯文本描述生成创意图片

```python
# 调用方式
wanx_image_gen.py --prompt "一张温馨的全家福照片" --output family.png
```

**适用场景**：
- 创意合成
- 背景替换
- 氛围重建

### 3. 自动图片压缩

**技术保障**：处理过大图片时自动优化

- 限制：5MB / 2048px
- 方式：用户无感知，透明压缩
- 目的：确保 API 调用稳定

## Prompt / 工作流 / 配置说明

### 核心提示词架构

#### 1. 诊断提示词模板

```
你是一个专业的废片诊断专家。请分析这张照片的问题：

问题类型（可能多选）：
- 路人干扰：背景出现陌生人物
- 姿势问题：人物姿态/表情不自然
- 服装问题：衣服颜色/款式不满意
- 光影问题：曝光过度/不足
- 构图问题：画面比例/位置不佳
- 缺人问题：需要添加人物
- 模糊问题：脱焦/抖动导致模糊

请给出：
1. 主要问题（最影响照片质量的问题）
2. 次要问题（可后续处理）
3. 修复优先级建议
```

#### 2. 修复提示词模板

```
基于原图进行以下修改：
[具体修改描述]

要求：
- 保持原图的主要构图和人物身份
- 修改自然，不留痕迹
- 保持光线一致性
- 画质清晰无损
```

快速获取对应提示词：

| 口令 | 功能 | 输出 |
|------|------|------|

| `/face` | 修复表情 | 调整面部表情的提示词 |
| `/pose` | 调整姿势 | 改善人物姿态的提示词 |
| `/cloth` | 换衣服 | 更换服装的提示词 |
| `/light` | 调光影 | 改善光线效果的提示词 |
| `/crop` | 改构图 | 调整画面构图的提示词 |
| `/add` | 添加人物 | 合成人物的提示词 |
| `/sharp` | 修复模糊 | 提升清晰度的提示词 |

### 工作流示例

#### 场景 1：旅游打卡照被路人抢镜

**用户输入**：
> 这张照片背景有路人，能帮忙修一下吗？

**处理流程**：
```
1. 诊断：识别为"路人干扰"问题
2. 策略：优先清除路人，保留主体
3. 提示词生成：
   "移除背景中的路人，保持主体人物不变，
   背景保持自然连贯，无明显处理痕迹"
4. 调用 wan2.7 进行图生图修复
```

#### 场景 2：合影有人闭眼

**用户输入**：
> 帮我把这个人的眼睛睁开

**处理流程**：
```
1. 诊断：识别为"姿势问题-表情"
2. 策略：仅修改眼部区域
3. 提示词生成：
   "基于原图，将闭眼的姿态自然地睁开，
   保持面部其他特征不变，表情自然"
4. 调用 wan2.7 进行局部修复
```

**功能**：

- 自动图片压缩
- wan2.7 文生图/图生图

**关键代码片段**：

```python
# 内置 API Key（随机选择）

    "sk-5ff36ee4460b4385a426e34db4d70b0d",
    "sk-02791f4e5ceb44ca99322e78c03e5ec1"
]

def get_api_key():
    return random.choice(API_KEYS)

# 图生图（废片拯救核心）
def image_to_image(input_image, prompt, output_path, strength=0.7):
    # 自动压缩过大图片
    processed_image = prepare_image(input_image)
    
    # 编码图片
    with open(processed_image, "rb") as f:
        image_data = base64.b64encode(f.read()).decode()
    
    # 调用 wan2.7 API
    url = "https://dashscope.aliyuncs.com/api/v1/services/aigc/image2image/image-synthesis"
    payload = {
        "model": "wanx-v1",
        "input": {"image": image_data, "prompt": prompt},
        "parameters": {"strength": strength, "n": 1}
    }
    
    # 异步任务轮询
    task_id = submit_async_task(url, payload)
    image_url = poll_task_status(task_id)
    
    # 下载结果
    download_image(image_url, output_path)
```

## 常见问题

**Q：图片压缩会影响画质吗？**
> A：压缩过程采用渐进式质量调整（95→50），在保证 API 调用成功的前提下最大程度保留画质。用户无感知。

**Q：修改强度 strength 参数如何选择？**
> A：
> - 0.3-0.5：轻微调整，保留原图较多
> - 0.5-0.7：中等调整，平衡效果
> - 0.7-1.0：大幅修改，更接近目标描述

**Q：修复后人物会不像本人吗？**
> A：wan2.7 在图像编辑时会保持人物身份一致性，但建议在 prompt 中明确"保持原图中的人物特征"以获得最佳效果。
